\begin{abstract}
Named Data Networking (NDN) shifts security focus from address-based filtering to content-centric inspection at forwarding nodes. We present \textit{Hybrid-Sentinel}, a three-tier machine-learning firewall that processes short packet-sequence windows and outputs \texttt{ALLOW}, \texttt{FLAG}, or \texttt{BLOCK} decisions with per-tier audit traces. Tier-1 is a lightweight Random Forest gate; Tier-2 is a six-class CNN-GRU exported to ONNX; Tier-3 is a Mahalanobis one-class detector on neural embeddings for zero-day novelty. On a held-out test set of 14,219 sequences (814 benign, 13,405 attack), the full cascade achieves 100\% attack detection, 0.86\% benign false-positive rate, Tier-2 macro-F1 of 0.985, and 4.19\,ms average CPU inference latency---3.1$\times$ faster than a 13\,ms gradient-boosting baseline.
\end{abstract}

Named Data Networking, intrusion detection, cascade classifier, ONNX, anomaly detection, deep packet inspection

\section{Introduction}
Traditional signature-based intrusion detection systems (IDS) such as Snort excel at line-rate matching but fail on zero-day and obfuscated attacks. Deep learning approaches (e.g., Packet BERT) improve generalization but are too expensive to run on every packet. NDN architectures process discrete content units at routers, motivating a \textit{hybrid} design: a fast filter on most traffic and a deep model on suspicious windows only \cite{tcl-wiki}.

We implement this hybrid pattern in Round~18 (R18) of Hybrid-Sentinel, fixing critical flaws in Round~17 (R17) where benign traffic was excluded from Tier-2 training, causing 100\% benign false-positive rate.

\section{Related Work}
Snort and Suricata rely on deterministic pattern matching. CNN/RNN hybrids have been applied to flow-based IDS (e.g., CIC-IDS datasets). Our contribution is a \textit{confidence-gated cascade} with explicit benign class handling, shared 17-feature contract, ONNX deployment, and Tier-3 embedding anomaly detection aligned with production API logic.

\section{System Architecture}
Fig.~\ref{fig:arch} shows the runtime cascade. Each input is a window of 20 packets $\times$ 17 behavioral features (length, protocol flags, entropy, flow statistics). Tier-1 computes $P(\text{BENIGN})$ on a 51-dimensional window summary; if $P \geq 0.90$, the window is \texttt{ALLOW}ed. Otherwise Tier-2 classifies among six labels including \texttt{BENIGN}. High-confidence attacks are \texttt{BLOCK}ed; medium confidence yields \texttt{FLAG}. Tier-3 scores embedding novelty and \texttt{FLAG}s anomalies.

Fig.~\ref{fig:orch} illustrates the build orchestration: PCAP $\rightarrow$ feature extraction $\rightarrow$ v6 sequences $\rightarrow$ training $\rightarrow$ ONNX export $\rightarrow$ unified \texttt{cascade\_r18.py} $\rightarrow$ FastAPI service.

\section{Methodology}

\subsection{Feature Extraction}
Seventeen features are extracted per packet from lab PCAP captures using Scapy: packet length, TCP/UDP/ICMP indicators, payload length and entropy, TCP flags, ports, TTL, and flow aggregates. Windows use size 20 and stride 10.

\subsection{Data Splitting}
Sequences are grouped by pseudo-flow blocks (1,000 contiguous same-label packets). A group-disjoint 70/15/15 train/validation/test split prevents temporal leakage. The scaler is fit on training data only. R17 incorrectly used the test set as validation; R18 corrects this.

\subsection{Tier-1 Gate}
A 150-tree Random Forest (156\,KB) performs binary BENIGN vs.\ ATTACK classification on concatenated mean, standard deviation, and last-packet features.

\subsection{Tier-2 CNN-GRU}
Architecture: Conv1D(17$\rightarrow$64) $\rightarrow$ GRU(64$\rightarrow$128, 2 layers) $\rightarrow$ linear classifier. Focal loss ($\gamma{=}2$) with elevated weight on \texttt{BENIGN}. Temperature scaling on the validation set calibrates confidence thresholds: BLOCK $>$ 0.95, FLAG $\in [0.80, 0.95]$.

\subsection{Tier-3 Zero-Day Net}
Mahalanobis distance on 128-D GRU embeddings fit to benign training distribution. Tier-3 outputs \texttt{FLAG} with label \texttt{ANOMALY}; it does not predict attack type.

\subsection{Attack Classes}
Five attack types plus benign: \texttt{BRUTE\_FORCE}, \texttt{DDOS\_HTTP\_FLOOD}, \texttt{SLOW\_HTTP}, \texttt{PORT\_SCAN}, \texttt{DNS\_TUNNELING}, \texttt{BENIGN}.

\section{Experimental Setup}
Traffic was captured in an isolated Docker lab emulating NDN producer--consumer patterns (HTTP, SSH, DNS services). The test set contains 14,219 sequences never seen during training. Evaluation uses \texttt{measure\_cascade\_flow.py} for end-to-end cascade metrics. Latency was benchmarked over 2,000 iterations on CPU with ONNX Runtime for Tier-2.

\section{Discussion and Limitations}
The system is validated on lab PCAP traffic (IP-overlay emulation), not yet on live NFD forwarders. Payload-byte models (Packet BERT) and malware-in-NDN-Data samples are planned integration paths. TLS-heavy enterprise traffic limits payload-dependent detection without edge decryption.

\section{Conclusion}
Hybrid-Sentinel demonstrates that a three-tier confidence-gated cascade can deliver high detection accuracy, low benign false-positive rate, and millisecond-scale CPU inference suitable for NDN edge deployment. R18 establishes a reproducible pipeline with unified configuration, single cascade implementation, and ONNX production path.
